# Implementation note: external and generated inventory

## Context

This note records the on-disk trees that were explicitly classified as generated, local-environment, third-party, or artifact storage during the 2026-04-01 review. They were inventoried for risk and scope control, not treated as first-class authored source.

Current review boundary reminders:

## Inventory method

The inventory pass used:

- `find ... -type f | wc -l`
- `du -sh`
- representative file sampling
- origin classification against root `.gitignore` and `internal_kb_fullstack/.gitignore`

Relevant ignore rules:

- root `.gitignore:4-14` ignores `.venv*`, backend caches, `node_modules`, `.next`, `.playwright-cli`, and `sample-data`
- `internal_kb_fullstack/.gitignore:1-12` ignores runtime and build outputs inside the app subtree

## Observations

### Generated counts are not additive as unique logical content

- `.next` can contain compiled output that packages or mirrors dependency material
- inventory counts should therefore not be added together as if they represented unique authored logic volume

### The current tree includes both authored deltas and artifact noise

- authored untracked review content and helper code must stay in the main review scope
- `output/` and tool caches must stay out of authored behavior conclusions

### Multiple local Python environments remain present

- `.venv` and `.venv312` coexist
- this is not a correctness bug by itself, but it creates operational ambiguity if contributors rely on different interpreters or package sets

## Impact

Coverage is explicit:

- authored code and docs were reviewed as source-bearing content
- large generated or third-party trees were classified and sampled only
- local corpus material was treated as ontology context rather than as tracked source

This separation keeps the repository-wide review honest about what was actually read and what was only inventoried.
